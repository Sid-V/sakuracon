## Goal

The goal is to create a simple to use interface where a user can access through a website that is hosted on vercel. The goal is for the user to be able to view all the events of the con and its details and they can favorite the events that they want to attend. The user should be able to sort by the keys provided in the schedule.md. There are 2 main objectives - 1. present the events in a easy to view manner which is only going to accessed through a broswer on a mobile device, 2. favorite the events that they want to attend and have it show up in a separate tab/page. 

Favorited events:
They should appear in sorted order by start time. Once the event is complete they should not show up in the favorited tab.

User log in and auth is not required. We can assume this will only be used by a group of my friends and you can store a user token in the browser cookies that can be used as a way to tell what events they favorited. Use the frontend-design skill as much as necessary.

## Tech stack
Hosting: Vercel
Frontend: NextJs

## Note
Do not ever run a dev server, always assume there is already one running and just tell the user to check.
